Requisitos Funcionales (Lo que la web debe hacer)
    - Catálogo Dinámico: Visualización de productos con filtros por marca y rubro.

    - Lógica de Precios Dual: * Público: Precio de lista + Mark Up + IVA.

    - Mayorista: Precio de lista - Bonificación (Sin IVA).

    - Sistema de Stock "Semáforo": Visualización gráfica (6 niveles) según parámetros a definir.

    - Gestión de Usuarios: Roles diferenciados (Admin, Mayorista, Minorista).

    - Flujo Híbrido de Pedidos: Carrito de compra directa y botón de "Solicitar Presupuesto" para obras/pesados.

    - Cotizador Descargable: Generación dinámica de un archivo (Excel/PDF) con precios actualizados para mayoristas.

    - Reseñas: Sistema de comentarios moderado en la ficha de producto.


Requisitos No Funcionales (lo que la web también va a incluir)
    - Sincronización: Importación masiva vía Excel/CSV (o API de Aleph si se puede).

    - Velocidad: Tiempo de carga bajo (especialmente en móviles) para no perder conversiones.

    - Seguridad: Autenticación robusta para el accesoo.


Herramientas de Marketing y Ventas (Para potenciar la conversión)
    - Tracking: Integración de Pixel de Meta y Google Analytics para medir el tráfico.

    - Retención: Sistema de Email Marketing y recuperación de carritos abandonados.

    - Contacto Directo: Botón flotante de WhatsApp en toda la web
